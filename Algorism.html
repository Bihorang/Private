Chapter 01 - 탐색과 자료구조 기반 탐색

1. 탐색 문제와 자료구조

탐색은 데이터 집합에서 원하는 값이나 조건을 만족하는 데이터를 찾는 작업이다. 같은 탐색 문제라도 데이터가 저장된 자료구조에 따라 가능한 알고리즘과 시간 복잡도가 달라진다. 시험에서는 "어떤 자료구조에서 왜 이 탐색이 유리한가"를 설명할 수 있어야 한다.

자료구조접근 특성적합한 탐색
배열인덱스로 임의 접근 가능선형 탐색, 정렬 시 이진 탐색
연결 리스트순차 접근 중심, 중간 원소 직접 접근 어려움선형 탐색
트리부모-자식 계층 구조DFS, BFS, BST 탐색
그래프정점과 간선의 연결 구조BFS, DFS
해시 테이블key를 주소로 변환평균적으로 빠른 key 탐색

자료구조가 탐색을 제한하는 이유는 저장 방식이 접근 방식을 결정하기 때문이다. 이진 탐색은 정렬된 데이터라는 조건뿐 아니라 middle 위치에 바로 접근할 수 있어야 한다. 그래서 정렬된 연결 리스트에서는 가운데 원소를 찾는 비용이 커져 이진 탐색의 장점이 사라진다.

2. 선형 탐색

선형 탐색은 처음부터 끝까지 순서대로 target과 비교하는 방식이다. 정렬 여부와 무관하게 사용할 수 있고, 배열, 연결 리스트, 스택, 큐, 정렬되지 않은 데이터에서 모두 가능하다.

  • 1차원 배열: 한 번의 반복으로 모든 원소 확인, 시간 복잡도 O(n).
  • 2차원 배열: 행과 열을 순회하므로 원소 수가 n x n이면 O(n^2).
  • 트리나 힙을 배열로 저장해도 선형 탐색은 가능하지만, 힙이나 BST의 구조적 장점을 쓰지 못한다.
  • 최선의 경우는 첫 원소에서 찾는 O(1), 최악은 마지막 또는 없음으로 O(n), 평균도 보통 O(n)으로 본다.

작동 단계:

  1. 탐색할 데이터의 첫 번째 원소에서 시작한다.
  2. 현재 원소와 target을 비교한다.
  3. 같으면 현재 위치를 반환하고 탐색을 종료한다.
  4. 다르면 다음 원소로 이동해 같은 비교를 반복한다.
  5. 마지막 원소까지 비교했는데 찾지 못하면 탐색 실패를 반환한다.

중간 결과로는 "현재 몇 번째 원소를 비교하고 있는가"가 중요하다. 시험에서 선형 탐색 과정을 쓰라고 하면 비교 순서와 성공/실패 종료 조건을 함께 적으면 된다.

3. 이진 탐색

이진 탐색은 정렬된 배열에서 중간값을 기준으로 탐색 범위를 절반씩 줄이는 알고리즘이다. 포인터 의미를 정확히 쓰는 것이 중요하다.

  • low: 현재 탐색 구간의 시작 인덱스.
  • high: 현재 탐색 구간의 끝 인덱스.
  • middle: (low + high) // 2.
  • target == A[middle]: 탐색 성공.
  • target > A[middle]: 오른쪽 절반 탐색, low = middle + 1.
  • target < A[middle]: 왼쪽 절반 탐색, high = middle - 1.

시간 복잡도는 매 단계 탐색 범위가 절반으로 줄어들기 때문에 2^k = n, k = log2 n이 되어 O(log n)이다. 단, 데이터가 정렬되어 있지 않다면 정렬 비용 O(n log n)이 먼저 필요하다. 한 번만 찾을 때는 정렬 비용 때문에 선형 탐색이 나을 수 있고, 같은 데이터에서 여러 번 찾을 때는 정렬 후 이진 탐색이 유리해진다.

작동 단계(오름차순 배열 기준):

  1. 배열이 오름차순 또는 내림차순으로 정렬되어 있는지 확인한다.
  2. low를 첫 인덱스, high를 마지막 인덱스로 둔다.
  3. middle = (low + high) // 2를 계산한다.
  4. A[middle]과 target을 비교한다.
  5. target이 A[middle]과 같으면 성공이다.
  6. target이 더 크면 왼쪽 구간은 버리고 low = middle + 1로 갱신한다.
  7. target이 더 작으면 오른쪽 구간은 버리고 high = middle - 1로 갱신한다.
  8. low > high가 되면 더 이상 탐색할 구간이 없으므로 실패다.

중간 결과는 매 반복의 low, high, middle, 남은 탐색 구간이다. 답안에서는 "왜 절반을 버릴 수 있는가"를 정렬 조건과 연결해 설명해야 한다.

내림차순 배열에서는 6~7단계의 비교 방향이 반대가 된다. 시험에서는 보통 오름차순 정렬 배열을 기준으로 설명하면 된다.

4. DFS, BFS, BST

DFS는 깊이 우선 탐색이다. 한 갈래를 끝까지 내려간 뒤 되돌아오며, 재귀나 스택을 사용한다. BFS는 너비 우선 탐색이다. 시작점에서 가까운 정점부터 레벨 순서로 방문하며, 큐를 사용한다.

구분DFSBFS
핵심 방향한 경로를 깊게가까운 레벨부터
자료구조Stack, recursionQueue
대표 용도cycle 탐지, 위상 정렬, 경로 탐색비가중 최단 경로, 레벨 탐색
주의점그래프에서는 방문 표시 필요그래프에서는 방문 표시 필요

BST는 Binary Search Tree다. 모든 노드에 대해 왼쪽 서브트리의 값은 현재 노드보다 작고, 오른쪽 서브트리의 값은 현재 노드보다 크다. BST 검증은 루트와 자식 하나만 보는 것이 아니라 모든 서브트리 범위 조건을 확인해야 한다. 균형이 잡힌 BST는 높이가 log n이므로 탐색이 O(log n)이고, 한쪽으로 편향되면 연결 리스트처럼 되어 O(n)까지 나빠진다.

BFS 작동 단계

  1. 시작 정점을 방문 처리하고 queue에 넣는다.
  2. queue에서 가장 먼저 들어온 정점을 꺼낸다.
  3. 꺼낸 정점의 인접 정점을 확인한다.
  4. 아직 방문하지 않은 인접 정점을 방문 처리하고 queue에 넣는다.
  5. queue가 빌 때까지 2~4단계를 반복한다.
  6. 비가중 그래프에서는 처음 발견된 거리 d가 시작점에서 해당 정점까지의 최단 간선 수가 된다.

DFS 작동 단계

  1. 시작 정점을 방문 처리한다.
  2. 현재 정점에서 아직 방문하지 않은 인접 정점 하나를 선택한다.
  3. 선택한 정점으로 깊게 이동해 같은 과정을 반복한다.
  4. 더 이상 갈 수 있는 인접 정점이 없으면 이전 정점으로 되돌아간다.
  5. 모든 reachable 정점을 방문할 때까지 반복한다.
  6. 재귀 호출 또는 stack의 push/pop 순서가 실제 방문 순서를 결정한다.

BST 탐색 작동 단계

  1. 루트 노드에서 시작한다.
  2. target이 현재 노드 값과 같으면 탐색 성공이다.
  3. target이 현재 노드보다 작으면 왼쪽 서브트리로 이동한다.
  4. target이 현재 노드보다 크면 오른쪽 서브트리로 이동한다.
  5. 이동할 자식이 없으면 target이 트리에 없으므로 실패다.
  6. 균형 BST에서는 한 단계마다 탐색 후보가 대략 절반으로 줄지만, 편향 BST에서는 한 줄로 내려가므로 O(n)이 된다.

5. 시험 주의점

  • 이진 탐색은 "정렬"과 "임의 접근"이 모두 필요하다.
  • 연결 리스트는 정렬되어 있어도 가운데로 바로 갈 수 없으므로 이진 탐색의 효과가 약하다.
  • BST는 트리이지만 균형이 깨지면 탐색이 느려진다.
  • DFS/BFS는 트리뿐 아니라 그래프에도 적용되며, 그래프에서는 방문 표시가 필수다.

Chapter 02 - Greedy 알고리즘과 최적화 전략

1. Greedy의 정의와 특징

Greedy 알고리즘은 매 단계에서 현재 가장 좋아 보이는 선택을 하고 그 선택을 되돌리지 않는 전략이다. 미래의 모든 경우를 계산하지 않으므로 보통 빠르고 공간 사용이 적다. 하지만 현재의 지역 최적해가 전체 최적해와 항상 같지는 않다.

Greedy 답안에서 "현재 최선 선택"만 쓰면 부족하다. 시험에서는 어떤 기준으로 선택하는지, 그 기준이 왜 전체 최적해를 해치지 않는지, 반례가 없는지까지 설명해야 한다.

2. 적용 조건

Greedy가 정답을 보장하려면 다음 성질을 확인해야 한다.

조건의미Greedy에서 하는 역할

Greedy 일반 작동 단계

  1. 문제에서 최적화 목표를 정한다. 예를 들어 동전 개수 최소화, 활동 개수 최대화, 총 가치 최대화처럼 목표가 분명해야 한다.
  2. 매 단계에서 무엇을 "현재 최선"으로 볼지 선택 기준을 정의한다.
  3. 후보들을 선택 기준에 맞게 정렬하거나 priority queue에 넣는다.
  4. 가장 좋아 보이는 후보를 하나 선택한다.
  5. 그 후보가 문제의 제약 조건을 위반하지 않으면 해에 포함한다.
  6. 선택한 후보를 되돌리지 않고 남은 부분 문제를 같은 기준으로 처리한다.
  7. 더 이상 선택할 후보가 없거나 목표를 달성하면 종료한다.

Greedy 답안의 핵심은 2단계의 선택 기준이다. "그리디를 쓴다"가 아니라 "종료 시간이 가장 빠른 활동을 고른다", "단위 무게당 가치가 큰 물건부터 고른다"처럼 기준을 구체적으로 써야 한다.

3. 대표 예시

Coin Change에서 큰 동전부터 선택하는 전략은 화폐 체계에 따라 성공하거나 실패한다. 500/100처럼 일반적인 체계에서는 자연스럽지만, 500/400/100으로 800원을 만들면 Greedy는 500+100+100+100으로 4개를 고르고, 최적해는 400+400으로 2개다. 이 예시는 Greedy Choice Property가 깨진 대표 반례다.

Activity Selection은 종료 시간이 가장 빠른 활동을 먼저 고르는 Greedy 전략이 핵심이다. 빨리 끝나는 활동을 고르면 남은 시간 구간이 넓어져 이후 선택 기회가 최대화된다. 시험에서는 "정렬 기준: 종료 시간 오름차순"과 "호환 가능한 활동만 선택"을 써야 한다.

Fractional Knapsack은 물건을 쪼갤 수 있기 때문에 단위 무게당 가치가 높은 순서로 담는 Greedy가 성립한다. 반대로 0/1 Knapsack은 물건을 쪼갤 수 없어서 단위 가치만 보고 고르면 반례가 생기며, DP가 필요하다.

Huffman Coding은 빈도 높은 문자에 짧은 코드를 부여하고 빈도 낮은 문자들을 깊은 쪽으로 결합하는 Greedy 트리 구성이다. 빈도 기반 우선순위 큐를 사용해 가장 빈도가 낮은 두 노드를 반복적으로 합친다.

Lambda 정렬은 알고리즘 자체가 아니라 선택 기준을 코드로 지정하는 구현 도구다. 예를 들어 단위 가치, 종료 시간, 빈도수 같은 값을 정렬 key로 줄 수 있다.

4. DP의 조건

DP를 적용하려면 두 조건이 중요하다.

  • Optimal Substructure: 전체 최적해가 부분 문제의 최적해로 구성된다.
  • Overlapping Subproblems: 같은 하위 문제가 여러 번 반복 등장한다.

최적 부분 구조가 없으면 부분 문제를 풀어도 전체 최적해로 이어지지 않는다. 중복 부분 문제가 없으면 값을 저장해도 재사용 이득이 작다. Greedy는 최적 부분 구조와 Greedy Choice Property를 보고, DP는 최적 부분 구조와 중복 부분 문제를 본다는 비교가 중요하다.

5. 구현 방식

Top-down DP는 재귀와 memoization을 사용한다. 필요한 상태를 호출하면서 계산하고, 이미 계산한 값은 memo에서 꺼내 재사용한다. Bottom-up DP는 작은 상태부터 반복문으로 table을 채운다. 모든 하위 상태가 먼저 계산되도록 순서를 잡아야 한다.

구분Top-downBottom-up
구현재귀 + memoization반복문 + tabulation
계산 상태필요한 상태만 계산 가능보통 작은 상태부터 전체 채움
장점문제 정의와 비슷함호출 오버헤드가 적고 순서 명확
주의점재귀 깊이, base casetable 초기화, 채우는 순서

Base Case는 더 작은 문제로 쪼갤 수 없는 시작 조건이다. 팩토리얼 예시에서 0! = 1 또는 1! = 1이 base case가 되며, 피보나치에서는 F(0), F(1)이 base case가 된다. Base Case가 틀리면 table 전체가 틀어진다.

DP 일반 작동 단계

  1. 상태를 정의한다. 예를 들어 dp[i][w], dp[i][j], dp[x]가 무엇을 의미하는지 먼저 정한다.
  2. Base Case를 정한다. 빈 문자열, 용량 0, 금액 0처럼 가장 작은 문제의 답을 채운다.
  3. 점화식을 세운다. 현재 상태를 더 작은 상태들의 답으로 표현한다.
  4. 계산 순서를 정한다. 필요한 하위 상태가 먼저 계산되도록 top-down 또는 bottom-up 순서를 잡는다.
  5. table 또는 memo를 채운다.
  6. 마지막 상태에서 최적값을 읽는다.
  7. 실제 선택 결과가 필요하면 predecessor, choice table, backtracking으로 해를 복원한다.

시험에서 DP 과정을 설명할 때는 "상태 정의 -> base case -> 점화식 -> table 채우기 -> 결과 읽기" 순서를 그대로 쓰면 안정적이다.

6. 성능 분석과 탐색 기법 연결

Brute Force는 모든 경우를 보므로 정답을 보장하지만 경우의 수가 팩토리얼 또는 지수적으로 커질 수 있다. 비밀번호 공격처럼 가능한 문자열 조합이 폭발하는 문제는 입력 크기 증가에 매우 민감하다.

Backtracking과 Branch and Bound는 상태 공간 트리에서 불필요한 가지를 제거해 탐색량을 줄인다. DP는 가지를 자르는 것보다 같은 계산을 저장해 다시 계산하지 않는 방식이다. 시험에서는 "가지치기"와 "저장/재사용"을 구분해서 써야 한다.

7. DP 응용

Coin Change 일반형은 금액별 최소 동전 수를 저장한다. dp[x]를 금액 x를 만들기 위한 최소 동전 수로 두고, 각 동전을 마지막에 하나 사용한 경우를 비교한다. 500/400/100, 800원 반례에서 DP는 dp[800] = min(dp[300]+1, dp[400]+1, dp[700]+1)처럼 여러 후보를 비교해 400+400을 찾을 수 있다.

0/1 Knapsack은 물건을 선택하거나 선택하지 않는 두 경우를 비교한다. 상태는 보통 dp[i][w]: 앞의 i개 물건만 고려하고 용량이 w일 때의 최대 가치로 정의한다. 물건 i를 넣을 수 없다면 이전 값 dp[i-1][w]를 가져오고, 넣을 수 있다면 max(선택 안 함, 선택함)을 비교한다.

Activity Selection도 DP 관점으로 보면 가능한 활동 조합을 비교할 수 있지만, 종료 시간이 빠른 활동을 고르는 Greedy가 조건을 만족하므로 더 간단하다. 이처럼 같은 문제를 Greedy와 DP 관점에서 비교하는 문제가 나올 수 있다.

알고리즘단계별 작동
Coin Change DP1. dp[x]를 금액 x를 만드는 최소 동전 수로 정의한다.

3. 금액 x를 작은 값부터 증가시키며 각 동전 c를 확인한다.

4. 가능성이 있으면 더 깊게 탐색한다.

3. 가능성이 있는 후보만 확장한다.
4. 더 좋은 해를 찾으면 현재 최적해를 갱신한다.
5. 남은 후보가 없으면 최종 최적해를 반환한다.

Chapter 04 - Sequence Alignment와 DP 응용

1. LCS에서 출발하는 서열 유사도

LCS는 Longest Common Subsequence다. 두 문자열에서 순서는 유지하지만 연속일 필요는 없는 공통 부분 수열 중 가장 긴 것을 찾는다. 예를 들어 X=ACGTA, Y=AGTTA라면 LCS로 AGTA를 생각할 수 있다.

LCS는 문자열 유사도 측정의 기본 DP 예시다. 순서가 보존되는 공통 구조를 찾기 때문에 생명정보학의 sequence alignment와 연결된다.

2. DP 테이블 정의와 점화식

dp[i][j]를 X의 앞 i글자와 Y의 앞 j글자의 LCS 길이로 둔다. 0행과 0열은 한 문자열이 비어 있는 경우이므로 0으로 초기화한다.

  • 문자가 같을 때: X[i-1] == Y[j-1]이면 dp[i][j] = dp[i-1][j-1] + 1.
  • 문자가 다를 때: dp[i][j] = max(dp[i-1][j], dp[i][j-1]).
  • 테이블은 왼쪽, 위쪽, 대각선 값이 먼저 계산되도록 행 또는 열 순서로 채운다.

시험에서는 table 값 자체보다 dp[i][j]가 무엇을 의미하는지, 왜 일치하면 대각선 +1인지, 불일치하면 위/왼쪽 중 큰 값을 고르는지 설명하는 것이 중요하다.

3. Backtracking

테이블을 채우는 단계는 최적해의 길이 또는 점수를 계산하는 단계다. 실제 LCS 문자열이나 alignment를 복원하려면 backtracking이 필요하다.

  • 우하단에서 시작한다.
  • 문자가 같으면 대각선으로 이동하고 해당 문자를 결과에 추가한다.
  • 문자가 다르면 위쪽과 왼쪽 중 더 큰 값 방향으로 이동한다.
  • 역방향으로 모은 결과를 뒤집어 최종 LCS를 얻는다.

즉 "최적해 값 계산"과 "최적해 구성 복원"은 별도 단계다.

LCS 작동 단계

  1. 두 문자열 X, Y의 길이를 각각 m, n이라 둔다.
  2. (m+1) x (n+1) 크기의 DP table을 만든다.
  3. 0행과 0열을 0으로 초기화한다.
  4. i=1..m, j=1..n 순서로 각 칸을 채운다.
  5. X[i-1] == Y[j-1]이면 대각선 값에 1을 더한다.
  6. 다르면 위쪽 값과 왼쪽 값 중 큰 값을 가져온다.
  7. dp[m][n]이 LCS 길이다.
  8. 실제 LCS 문자열은 우하단에서 backtracking하여 복원한다.

4. Global Alignment

Needleman-Wunsch는 두 서열 전체를 처음부터 끝까지 맞추는 global alignment 알고리즘이다. 전체 유사도, 진화적 거리, 전체 서열 비교에 적합하다. Match, mismatch, gap penalty를 점수 체계로 사용한다.

Global alignment는 전체를 맞추기 때문에 한쪽 서열에 긴 삽입/삭제가 있거나 특정 부분만 유사할 때는 불필요한 gap penalty가 커질 수 있다. 점수 체계가 결과에 큰 영향을 준다는 점도 중요하다.

Needleman-Wunsch 작동 단계

  1. 두 서열 전체를 비교할 DP table을 만든다.
  2. 0행과 0열을 gap penalty가 누적되도록 초기화한다.
  3. 각 칸에서 세 후보를 계산한다: 대각선 이동(match/mismatch), 위쪽 이동(gap), 왼쪽 이동(gap).
  4. scoring rule에 따라 세 후보 중 가장 좋은 점수를 선택한다.
  5. table을 끝까지 채운 뒤 우하단 값이 전체 서열 정렬의 최적 점수다.
  6. 우하단에서 시작해 선택된 방향을 따라 backtracking한다.
  7. 대각선은 두 문자를 맞춘 것이고, 위/왼쪽 이동은 gap을 넣은 것이다.
  8. 좌상단에 도달하면 global alignment가 완성된다.

5. Local Alignment

Smith-Waterman은 local alignment 알고리즘이다. 전체 서열이 아니라 가장 유사한 부분 구간을 찾는다. 핵심은 음수 점수를 0으로 되돌리는 0 clamp다.

  • 테이블 값은 max(0, 대각선+match/mismatch, 위+gap, 왼쪽+gap)처럼 계산된다.
  • backtracking은 우하단이 아니라 최고 점수 위치에서 시작한다.
  • 0을 만나면 해당 local alignment가 끝난다.
  • motif, domain, 특정 패턴 탐색에 적합하다.

Smith-Waterman 작동 단계

  1. 두 서열을 비교할 DP table을 만들고 0행과 0열을 0으로 초기화한다.
  2. 각 칸에서 대각선(match/mismatch), 위쪽(gap), 왼쪽(gap), 0을 후보로 둔다.
  3. 네 후보 중 가장 큰 값을 현재 칸에 채운다.
  4. 음수 점수는 0으로 잘리므로 불리한 정렬 구간은 이어가지 않는다.
  5. table 전체에서 가장 큰 점수를 찾는다.
  6. 최고 점수 칸에서 backtracking을 시작한다.
  7. 0을 만나면 local alignment가 끝난다.
  8. backtracking으로 얻은 구간이 가장 유사한 부분 서열 정렬이다.

7. 실제 생명정보학 연결

DP 기반 alignment는 보통 시간 복잡도와 공간 복잡도가 O(mn)이다. 두 서열 길이가 길어지면 table이 매우 커진다. 유전체 수준의 대규모 데이터에서는 정확한 DP가 부담되므로 BLAST, FASTA 같은 heuristic 알고리즘을 사용한다.

Heuristic은 최적해 보장을 일부 포기하고 빠른 후보 탐색을 우선한다. 시험에서는 DP는 정확한 기준 알고리즘, heuristic은 실무 대규모 처리의 절충안으로 구분하면 된다.

Chapter 05 - Graph Elementary와 BFS DFS

1. 그래프 기본 정의

그래프는 G = (V, E)로 표현한다. V는 vertex 또는 node의 집합이고, E는 edge의 집합이다. 지하철역과 노선, 전기회로, SNS 친구 관계, 웹 페이지 링크처럼 객체 사이의 관계를 표현할 때 사용한다. 트리도 cycle이 없는 연결 그래프로 볼 수 있다.

쾨니히스베르크 다리 문제는 그래프 이론의 역사적 출발점으로, 정점과 간선을 이용해 실제 문제를 추상화하는 사고를 보여준다.

2. 그래프 종류와 수식

  • 무방향 그래프: 간선에 방향이 없으며 상호 관계를 나타낸다.
  • 방향 그래프: 간선에 방향이 있으며 일방통행, 웹 링크 등에 적합하다.
  • 자기 루프: 정점이 자기 자신으로 연결된 간선.
  • 완전 그래프: 가능한 모든 정점 쌍이 연결된 그래프.
  • 가중치 그래프: 간선에 비용, 거리, 시간, 강도 같은 값이 붙은 그래프.
  • 부분 그래프: 원래 그래프의 정점과 간선 일부로 만든 그래프.

정점 수가 n일 때 자기 루프가 없다고 하면 무방향 그래프의 최대 간선 수는 n(n-1)/2, 방향 그래프의 최대 간선 수는 n(n-1)이다. 무방향은 두 정점 쌍을 한 번만 세고, 방향은 u -> v와 v -> u를 다르게 세기 때문이다.

3. 그래프 용어

  • 인접: 두 정점이 간선으로 직접 연결된 상태.
  • 차수: 정점에 연결된 간선 수.
  • 진입 차수: 방향 그래프에서 해당 정점으로 들어오는 간선 수.
  • 진출 차수: 해당 정점에서 나가는 간선 수.
  • 경로: 간선을 따라 이동하는 정점들의 리스트.
  • 경로 길이: 비가중 그래프에서는 간선 수, 가중 그래프에서는 가중치 합으로 해석한다.

5. BFS

BFS는 Breadth-First Search로, 시작 정점에서 가까운 정점부터 방문한다. Queue를 사용하며 FIFO 순서가 핵심이다.

  • WHITE: 아직 발견되지 않은 정점.
  • GRAY: 발견되었지만 인접 정점 처리가 남은 정점.
  • BLACK: 인접 정점 처리가 끝난 정점.
  • d[v]: 시작점에서 v까지의 거리, 비가중 그래프에서는 최단 간선 수.
  • pi[v]: BFS tree에서 v의 부모.

BFS는 가중치가 없는 그래프에서 시작점 기준 최단 간선 수를 계산한다. 다만 인접 정점이 queue에 들어오는 순서에 따라 BFS tree의 모양은 달라질 수 있다. 거리 값은 최단 거리로 유지되지만 부모 선택은 순서에 영향을 받을 수 있다.

BFS 상세 작동 단계

  1. 모든 정점을 WHITE, 거리 ∞, 부모 NIL로 초기화한다.
  2. 시작 정점 s를 GRAY로 바꾸고 d[s] = 0으로 둔다.
  3. 시작 정점을 queue에 넣는다.
  4. queue에서 정점 u를 꺼낸다.
  5. u의 모든 인접 정점 v를 확인한다.
  6. v가 WHITE이면 GRAY로 바꾸고 d[v] = d[u] + 1, pi[v] = u로 갱신한 뒤 queue에 넣는다.
  7. u의 인접 정점 처리가 끝나면 u를 BLACK으로 바꾼다.
  8. queue가 비면 종료한다.

6. DFS와 간선 분류

DFS는 Depth-First Search로, 한 경로를 깊게 들어간 뒤 더 이상 갈 곳이 없으면 되돌아온다. 재귀 또는 stack을 사용하며, discovery time과 finish time을 기록할 수 있다.

DFS 간선 분류는 방향 그래프에서 특히 중요하다.

  • Tree Edge: DFS tree를 구성하는 간선.
  • Back Edge: 현재 정점에서 조상 정점으로 돌아가는 간선. cycle 탐지와 연결된다.
  • Forward Edge: 조상에서 자손으로 가지만 tree edge는 아닌 간선.
  • Cross Edge: 서로 다른 DFS subtree 사이를 잇는 간선.

Back Edge가 있으면 cycle이 존재한다. DFS 종료 시간은 위상 정렬과도 연결된다.

DFS 상세 작동 단계

  1. 모든 정점을 WHITE, 부모 NIL로 초기화한다.
  2. 시간 변수 time = 0을 둔다.
  3. 아직 WHITE인 정점을 발견하면 DFS-Visit을 시작한다.
  4. 정점을 GRAY로 바꾸고 discovery time을 기록한다.
  5. 인접 정점을 하나씩 확인한다.
  6. 인접 정점이 WHITE이면 tree edge로 보고 부모를 현재 정점으로 설정한 뒤 재귀적으로 방문한다.
  7. 인접 정점이 GRAY이면 back edge이므로 cycle 가능성을 판단한다.
  8. 모든 인접 정점 처리가 끝나면 BLACK으로 바꾸고 finish time을 기록한다.
  9. 아직 방문하지 않은 정점이 있으면 새 DFS tree를 시작한다.

7. 후속 단원 연결

그래프 표현 방식은 MST, Dijkstra, Bellman-Ford, Floyd-Warshall의 구현 비용에 영향을 준다. BFS의 d와 pi는 Dijkstra의 거리 추정값과 predecessor로 확장된다. DFS의 cycle 판단은 MST에서 cycle을 만들지 않아야 한다는 조건과 이어진다.

Chapter 06 - MST Kruskal Prim

1. MST 정의

Spanning Tree는 그래프의 모든 정점을 포함하는 트리다. Tree는 연결되어 있고 cycle이 없는 그래프다. Minimum Spanning Tree는 가능한 spanning tree 중 간선 가중치 합이 최소인 트리다.

MST 조건은 함께 기억해야 한다.

  • 모든 정점을 연결한다.
  • cycle이 없다.
  • 간선 가중치 합이 최소다.
  • 정점이 V개이면 간선은 정확히 V-1개다.
  • E < V-1이면 모든 정점을 연결할 수 없다.
  • E > V-1이면 cycle이 생길 가능성이 있으므로 선택 간선을 관리해야 한다.

MST는 최단 경로 문제가 아니다. 목적은 시작점에서 각 정점까지의 거리 최소가 아니라, 모든 정점을 연결하는 전체 비용 최소화다.

2. Kruskal 알고리즘

Kruskal은 Greedy 기반 MST 알고리즘이며 간선 중심이다.

  1. 모든 간선을 가중치 오름차순으로 정렬한다.
  2. 가장 작은 간선부터 차례로 본다.
  3. 그 간선이 cycle을 만들지 않으면 선택한다.
  4. cycle을 만들면 버린다.
  5. 선택 간선 수가 V-1개가 되면 MST가 완성된다.

Cycle 검출에는 Union-Find를 사용한다.

  • Find: 정점이 속한 집합의 대표 root를 찾는다.
  • Union: 서로 다른 집합을 합친다.
  • 같은 root를 가진 두 정점을 잇는 간선은 cycle을 만들므로 선택하지 않는다.
  • Path Compression: Find 중 만난 노드의 부모를 root로 바로 바꿔 다음 탐색을 빠르게 한다.
  • Rank 또는 size 기반 union: 작은 트리를 큰 트리 아래에 붙여 높이를 줄인다.

Kruskal 작동 단계

  1. 각 정점을 자기 자신만 포함하는 독립 집합으로 초기화한다.
  2. 모든 간선을 가중치 오름차순으로 정렬한다.
  3. 가장 작은 간선부터 하나씩 확인한다.
  4. 간선 (u, v)의 두 정점에 대해 Find(u), Find(v)를 수행한다.
  5. 두 대표가 다르면 서로 다른 component를 잇는 간선이므로 MST에 포함한다.
  6. 선택한 간선의 두 집합을 Union(u, v)로 합친다.
  7. 두 대표가 같으면 이미 같은 component 안의 간선이므로 cycle을 만들기 때문에 버린다.
  8. 선택한 간선 수가 V-1개가 되면 종료한다.

중간 결과는 정렬된 간선 목록, 선택된 간선 집합, 각 정점의 union-find 대표다.

3. Prim 알고리즘

Prim도 Greedy 기반 MST 알고리즘이지만 노드/트리 확장 중심이다.

  1. 임의의 시작 정점을 선택한다.
  2. 현재 MST에 포함된 정점들과 외부 정점을 잇는 간선 중 최소 가중치 간선을 고른다.
  3. 그 간선과 외부 정점을 MST에 추가한다.
  4. 새 정점에서 나가는 후보 간선을 priority queue에 넣거나 갱신한다.
  5. 모든 정점이 포함될 때까지 반복한다.

Prim은 현재 트리에서 바깥으로 뻗는 최소 간선을 고르므로, 이미 트리에 포함된 정점으로 돌아가는 간선은 제외한다. 이 방식 때문에 cycle 방지가 자연스럽게 반영된다.

Prim 작동 단계

  1. 시작 정점을 하나 고르고 MST 집합에 넣는다.
  2. 시작 정점에서 나가는 간선을 priority queue에 넣는다.
  3. priority queue에서 가중치가 가장 작은 간선을 꺼낸다.
  4. 그 간선이 이미 MST 안에 있는 정점끼리 연결하면 버린다.
  5. 외부 정점을 연결하는 간선이면 그 간선과 새 정점을 MST에 추가한다.
  6. 새로 추가된 정점에서 나가는 간선 중 외부 정점으로 향하는 간선을 priority queue에 넣는다.
  7. 모든 정점이 MST에 포함될 때까지 반복한다.
  8. 선택된 간선 V-1개가 MST가 된다.

중간 결과는 현재 MST에 포함된 정점 집합, 후보 간선 priority queue, 선택된 간선 목록이다.

5. 응용

MST는 도시 간 도로 건설 비용 최소화, 네트워크 연결 비용 최소화, 세포 관계 분석, DNA fragment 연결, 단백질 상호작용 네트워크의 핵심 backbone 추출, signaling pathway 추출 등에 연결된다. 응용 문제에서는 "모든 대상 연결"과 "전체 비용 최소"가 보이면 MST를 의심한다.

1. 최단 경로 문제로 이어지는 이유

선형 자료구조는 복잡한 연결 관계나 경로 문제를 표현하기 어렵다. 그래프에서는 건물, 방, 교차로, 세포, 단백질 등을 vertex로 두고 연결 관계를 edge로 표현할 수 있다. 경로 길이는 비가중 그래프에서는 간선 수, 가중 그래프에서는 가중치 합이다.

  • 모든 쌍: Floyd-Warshall 또는 Johnson.

3. Dijkstra 핵심

Dijkstra는 비음수 가중치 그래프에서 하나의 시작점으로부터 모든 정점까지의 최단 거리를 구한다. Greedy 전략은 "아직 확정되지 않은 정점 중 시작점 기준 거리 추정값 d가 가장 작은 정점을 확정"하는 것이다.

초기화는 Initialize-Single-Source로 한다.

  • 시작점 s.d = 0.
  • 나머지 정점 v.d = ∞.
  • predecessor v.pi = NIL.

d[v]는 현재까지 발견한 시작점에서 v까지의 최단거리 추정값이다. Dijkstra는 비음수 조건 때문에 가장 작은 d를 가진 정점을 꺼내는 순간 그 값이 확정된다.

4. Relax

Relax는 더 짧은 경로를 발견했을 때 거리와 predecessor를 갱신하는 연산이다.

  • 조건: v.d > u.d + w(u,v).
  • 갱신: v.d = u.d + w(u,v).
  • predecessor 갱신: v.pi = u.

Priority Queue를 사용하면 d가 감소한 정점의 key를 낮추는 Decrease-Key가 필요하다. Python heapq처럼 decrease-key를 직접 제공하지 않는 경우 새 값을 다시 넣고 꺼낼 때 오래된 값을 무시하는 방식도 쓴다. 시험에서는 구현 세부보다 "거리 추정값을 줄이고 부모를 바꾼다"는 의미가 중요하다.

5. Dijkstra 작동 단계

  1. 모든 정점의 거리 d를 ∞, predecessor pi를 NIL로 초기화한다.
  2. 시작 정점 s의 거리를 0으로 둔다.
  3. 모든 정점을 거리 d 기준 min-priority queue에 넣는다.
  4. queue에서 d가 가장 작은 정점 u를 꺼내 최단 거리를 확정한다.
  5. u의 모든 인접 간선 (u, v)를 확인한다.
  6. v.d > u.d + w(u, v)이면 relax를 수행해 v.d와 v.pi를 갱신한다.
  7. 갱신된 거리 값을 priority queue에 반영한다.
  8. queue가 빌 때까지 4~7단계를 반복한다.
  9. 최종 d 값은 시작점에서 각 정점까지의 최단 거리이고, pi를 따라가면 최단 경로를 복원할 수 있다.

중간 결과는 매 반복에서 확정되는 정점, priority queue의 후보, 각 정점의 d와 pi다. 음수 간선이 있으면 4단계의 "확정"이 깨질 수 있다.

6. Priority Queue와 복잡도

Dijkstra는 min-priority queue 또는 min-heap으로 가장 작은 d 값을 가진 정점을 추출한다. 추출한 정점의 인접 간선을 모두 relax한다.

구현시간 복잡도설명

이진 힙 기반O(E log V)간선 relax와 heap 연산

그래프가 밀집인지 희소인지, decrease-key를 어떻게 처리하는지에 따라 실제 설명이 달라질 수 있다.

7. Dijkstra의 한계와 비교

Dijkstra는 음수 간선이 있으면 실패할 수 있다. 이미 확정한 정점의 거리가 나중에 음수 간선을 통해 더 작아질 수 있는데, Dijkstra는 확정된 정점을 다시 충분히 전파하지 않기 때문이다. 이 문제를 해결하려면 Bellman-Ford처럼 모든 간선을 반복적으로 relax해야 한다.

Chapter 08 - Bellman-Ford Floyd-Warshall Johnson

1. Bellman-Ford로 이어지는 이유

Dijkstra는 모든 간선이 비음수라는 조건이 필요하다. 음수 간선이 있으면 확정된 정점의 거리 값이 뒤늦게 더 작아질 수 있다. Bellman-Ford는 단일 시작점 최단 경로 문제에서 음수 간선을 허용하기 위해 모든 간선을 반복 relax한다.

음수 간선과 음수 사이클은 구분해야 한다. 음수 간선은 Bellman-Ford로 처리할 수 있지만, 시작점에서 도달 가능한 음수 사이클이 있으면 그 사이클을 계속 돌수록 경로 비용이 무한히 작아져 최단 경로가 정의되지 않는다.

2. Bellman-Ford 알고리즘

입력은 그래프 G, 가중치 함수 w, 시작 정점 s다.

  1. Initialize-Single-Source를 수행한다.
  2. 전체 간선을 |V|-1번 반복 relax한다.
  3. 한 번 더 전체 간선을 검사한다.
  4. 추가 relax가 가능하면 도달 가능한 음수 사이클이 존재하므로 실패를 반환한다.
  5. 추가 relax가 없으면 최단 거리와 predecessor가 유효하다.

|V|-1번 반복하는 이유는 cycle 없는 단순 경로가 가질 수 있는 최대 간선 수가 |V|-1이기 때문이다. 시간 복잡도는 O(VE)다. Dijkstra보다 느리지만 적용 범위가 넓고 음수 사이클을 탐지할 수 있다.

Bellman-Ford 작동 단계

  1. 모든 정점의 거리 d를 ∞, predecessor pi를 NIL로 초기화한다.
  2. 시작 정점 s.d = 0으로 둔다.
  3. 전체 간선 목록을 준비한다.
  4. 1번째 반복부터 |V|-1번째 반복까지 모든 간선을 순서대로 relax한다.
  5. 반복이 진행될수록 더 많은 간선을 사용하는 최단 경로 정보가 전파된다.
  6. |V|-1회 반복 후 전체 간선을 한 번 더 검사한다.
  7. 이 추가 검사에서 relax가 가능하면 도달 가능한 음수 사이클이 존재한다.
  8. 추가 relax가 없으면 현재 d와 pi가 유효한 최단 경로 결과다.

중간 결과는 반복 횟수별 distance table이다. 시험에서 Bellman-Ford 과정을 묻는다면 각 반복 후 어떤 정점의 거리가 갱신되었는지 표시하는 것이 핵심이다.

3. Bellman-Ford 중간 결과

Bellman-Ford의 시험형 핵심은 반복별 distance table이다. 첫 번째 반복 후에는 간선 1개 이하를 쓰는 최단 정보가 전파되고, 두 번째 반복 후에는 간선 2개 이하, 이런 식으로 |V|-1번째 반복까지 정보가 퍼진다.

Relax 조건과 갱신식은 Dijkstra와 같다.

  • 조건: v.d > u.d + w(u,v).
  • 갱신: v.d = u.d + w(u,v).
  • predecessor: v.pi = u.

|V|-1회 이후에도 어떤 간선 (u,v)에서 v.d > u.d + w(u,v)가 성립하면 음수 사이클이다.

4. DAG Shortest Paths

DAG는 Directed Acyclic Graph, 즉 방향성 있고 cycle이 없는 그래프다. DAG에서는 위상 정렬 후 그 순서대로 간선을 relax하면 최단 경로를 구할 수 있다.

  • 시간 복잡도: O(V+E).
  • 음수 간선이 있어도 cycle이 없으므로 처리 가능하다.
  • 일반 그래프의 Bellman-Ford보다 훨씬 빠를 수 있다.

조건이 특수하면 더 빠른 알고리즘을 선택할 수 있다는 점이 중요하다.

DAG Shortest Paths 작동 단계

  1. 그래프가 DAG인지, 즉 방향성이 있고 cycle이 없는지 확인한다.
  2. 정점들을 위상 정렬한다.
  3. 모든 정점 거리 d를 ∞, 시작 정점은 0으로 초기화한다.
  4. 위상 정렬 순서대로 정점 u를 하나씩 처리한다.
  5. u에서 나가는 모든 간선 (u, v)를 relax한다.
  6. 위상 순서상 뒤에 오는 정점으로 정보가 한 방향으로 전파된다.
  7. 모든 정점을 처리하면 최단 거리 계산이 끝난다.

5. APSP 정의와 행렬

APSP는 All-Pairs Shortest Paths로 모든 정점 쌍 (u,v)에 대해 최단 경로를 구하는 문제다. 출력은 보통 V x V distance matrix다. 경로 복원을 위해 predecessor matrix Pi를 함께 둘 수 있다.

초기 Weight Matrix는 다음처럼 둔다.

  • 자기 자신으로 가는 비용: 0.
  • 직접 간선이 있으면 해당 간선 가중치.
  • 직접 경로가 없으면 ∞.

SSSP를 모든 정점에서 반복하는 방법도 가능하다. 그러나 Dijkstra를 V번 반복하려면 음수 간선이 없어야 하고, Bellman-Ford를 V번 반복하면 매우 느리다. 밀집 그래프에서 Bellman-Ford 반복은 O(V^4) 수준까지 커질 수 있으므로 APSP 전용 알고리즘이 필요하다.

6. SLOW-APSP와 FASTER-APSP

SLOW-APSP와 FASTER-APSP는 간선 수 기반 DP로 볼 수 있다. L_ij^(r)는 최대 r개의 간선을 사용해 i에서 j까지 가는 최소 가중치다. EXTEND-SHORTEST-PATHS는 행렬 곱셈과 비슷하지만, 일반 곱셈의 sum/product 대신 min/plus를 사용한다.

  • SLOW-APSP: 사용할 수 있는 간선 수를 1개씩 늘림, Θ(n^4).
  • FASTER-APSP: 반복 제곱을 사용, Θ(n^3 lg n).

시험에서 핵심은 세부 코드보다 "최대 간선 수를 늘려가며 최단 경로 정보를 확장하는 DP"라는 관점이다.

SLOW-APSP / FASTER-APSP 작동 단계

  1. 초기 행렬 L^(1)을 edge weight matrix로 둔다.
  2. L_ij^(r)가 최대 r개 간선을 사용한 i -> j 최단거리라고 해석한다.
  3. EXTEND-SHORTEST-PATHS로 중간 정점 k를 거치는 모든 후보 L_ik + W_kj를 비교한다.
  4. SLOW-APSP는 사용할 수 있는 간선 수를 1개씩 늘려 n-1개까지 확장한다.
  5. FASTER-APSP는 반복 제곱처럼 1, 2, 4, 8개 간선 범위로 빠르게 확장한다.
  6. 마지막 행렬이 모든 정점 쌍의 최단거리 행렬이 된다.

7. Floyd-Warshall

Floyd-Warshall은 중간 정점 기반 DP다. d_ij^(k)를 중간 정점이 {1,...,k}에 속하는 경우에 i에서 j까지 가는 최단 경로 길이라고 정의한다.

  • 초기 조건: d_ij^(0) = w_ij.
  • 점화식: d_ij^(k) = min(d_ij^(k-1), d_ik^(k-1) + d_kj^(k-1)).
  • 의미: 정점 k를 경유하지 않는 경우와 k를 경유하는 경우 중 작은 값을 선택한다.
  • 구현: k, i, j의 3중 루프.
  • 시간 복잡도: Θ(n^3).
  • 공간 복잡도: matrix를 덮어쓰면 Θ(n^2)까지 최적화 가능.

Floyd-Warshall은 음수 간선을 허용하지만 음수 사이클이 있으면 최단 경로가 정의되지 않는다. 중간 결과는 D^(0), D^(1), ..., D^(n) matrix의 변화로 설명한다.

Floyd-Warshall 작동 단계

  1. 초기 distance matrix D를 만든다.
  2. D[i][i] = 0, 직접 간선이 있으면 그 가중치, 없으면 ∞로 초기화한다.
  3. 중간 정점으로 허용할 정점 k를 1번부터 n번까지 하나씩 늘린다.
  4. 각 k에 대해 모든 출발 정점 i와 도착 정점 j 쌍을 확인한다.
  5. 기존 거리 D[i][j]와 k를 거치는 거리 D[i][k] + D[k][j]를 비교한다.
  6. k를 거치는 경로가 더 짧으면 D[i][j]를 갱신한다.
  7. 모든 k를 처리하면 D[i][j]는 모든 정점 쌍의 최단거리다.
  8. 경로 복원이 필요하면 predecessor matrix도 함께 갱신한다.

중간 결과는 각 k 단계 이후의 distance matrix다. 답안에서는 "k를 경유하지 않는 경우 vs k를 경유하는 경우"의 비교를 반드시 써야 한다.

8. Johnson 알고리즘

Johnson은 희소 그래프에서 APSP를 효율적으로 구하기 위한 알고리즘이다. 음수 간선은 허용하지만 음수 사이클은 허용하지 않는다.

절차는 다음과 같다.

  1. 새 정점 s를 추가하고 모든 정점으로 가중치 0 간선을 연결한다.
  2. Bellman-Ford를 한 번 실행해 potential h(v)를 계산한다.
  3. w'(u,v) = w(u,v) + h(u) - h(v)로 reweighting하여 모든 간선을 비음수화한다.
  4. 각 정점을 시작점으로 Dijkstra를 실행한다.
  5. 원래 가중치 기준 거리로 복원한다.

핵심은 reweighting이 경로의 상대적 최단성은 보존하면서 Dijkstra를 사용할 수 있게 만드는 것이다. 밀집 그래프에서는 Floyd-Warshall이 단순하고, 희소 그래프에서는 Johnson이 유리할 수 있다.

Johnson 작동 단계

  1. 그래프에 새 정점 s를 추가한다.
  2. s에서 모든 기존 정점으로 가중치 0인 간선을 추가한다.
  3. Bellman-Ford를 s에서 실행해 potential h(v)를 구한다.
  4. Bellman-Ford가 음수 사이클을 발견하면 Johnson은 중단한다.
  5. 모든 간선 (u, v)의 가중치를 w'(u,v) = w(u,v) + h(u) - h(v)로 바꾼다.
  6. reweighting된 그래프에서는 모든 간선 가중치가 비음수가 된다.
  7. 각 정점을 시작점으로 Dijkstra를 실행한다.
  8. Dijkstra 결과를 원래 가중치 기준 거리로 되돌린다.
  9. 모든 시작점에 대한 결과를 모아 APSP distance matrix를 만든다.

중간 결과는 Bellman-Ford의 potential 값, reweighted edge, 각 시작점별 Dijkstra distance table이다.

3. 반드시 비교할 개념

DP vs Greedy

Greedy는 현재 최선 선택을 고정하고 되돌리지 않는다. DP는 가능한 선택을 비교하고 중간 결과를 저장한다. Greedy는 빠르지만 조건부이고, DP는 공간을 쓰지만 상태 정의가 맞으면 최적해를 체계적으로 구한다. 답안에는 Greedy Choice Property와 Overlapping Subproblems를 섞어 쓰지 않도록 주의한다.

Backtracking vs Branch and Bound vs DP

Backtracking은 조건을 어긴 가지를 되돌린다. Branch and Bound는 bound를 사용해 현재 최적보다 나쁜 후보를 제거한다. DP는 같은 하위 문제가 반복될 때 저장해 재사용한다. 셋 모두 탐색량을 줄이지만, Backtracking/Branch and Bound는 가지치기, DP는 저장과 재사용이 핵심이다.

BFS vs DFS

BFS는 queue로 가까운 정점부터 처리하고 비가중 최단 거리를 구할 수 있다. DFS는 stack 또는 재귀로 깊게 들어가며 discovery/finish time과 간선 분류에 강점이 있다.

Kruskal vs Prim

Kruskal은 간선을 전역 정렬하고 Union-Find로 cycle을 막는다. Prim은 현재 트리에서 외부로 나가는 최소 간선을 고르며 priority queue를 사용한다. 둘 다 MST를 구하지만 Kruskal은 간선 중심, Prim은 트리 확장 중심이다.

Dijkstra vs Bellman-Ford vs Floyd-Warshall

Dijkstra는 SSSP, 비음수 가중치, Greedy 확정 방식이다. Bellman-Ford는 SSSP, 음수 간선 허용, 반복 relax와 음수 사이클 탐지다. Floyd-Warshall은 APSP, 중간 정점 기반 DP, Θ(n^3)이다.

4. 시험형 답안 틀

정의형 답안은 정의 -> 조건 -> 예시 순서로 쓴다. 예를 들어 MST는 "모든 정점을 연결하고 cycle이 없으며 간선 수가 V-1이고 가중치 합이 최소인 spanning tree"라고 써야 한다.

비교형 답안은 목적 -> 조건 -> 자료구조 -> 복잡도 -> 한계 순서가 좋다. 예를 들어 Dijkstra와 Bellman-Ford 비교에서는 둘 다 SSSP지만 Dijkstra는 비음수 가중치에서 빠르고, Bellman-Ford는 느리지만 음수 간선과 음수 사이클 탐지가 가능하다고 쓴다.

과정형 답안은 "초기화, 반복 중 갱신, 종료 조건"을 써야 한다. DP는 base case와 점화식, 그래프 알고리즘은 queue/PQ/matrix의 변화, MST는 선택 간선과 cycle 여부를 제시한다.

시험 직전 3분 요약

  1. 탐색은 자료구조 조건부터 본다. 이진 탐색은 정렬과 임의 접근이 필요하다.
  2. Greedy는 선택 기준과 반례를 말하고, DP는 상태 정의와 점화식을 말한다.
  3. BFS는 queue와 비가중 최단거리, DFS는 stack/재귀와 cycle 판단이다.
  4. MST는 모든 정점 연결 최소 비용이고, 최단경로는 시작점 또는 모든 쌍의 거리 최소다.
  5. Dijkstra, Bellman-Ford, Floyd-Warshall은 가중치 조건과 문제 범위로 선택한다.
