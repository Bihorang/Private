week02: 알고리즘과 복잡도

개념 구조정리헷갈리기 쉬운 부분
알고리즘 정의주어진 문제를 해결하기 위해 논리 정연한 방법에 의해 계획한 해결 방안을 정형적이고 체계적으로 기술한 것. 잘 짜인 계산 순서를 가진 프로그래밍.코드 자체만이 아니라 "계산 순서"가 핵심.
알고리즘 요건입력과 출력의 명확성, 단계별 명령의 명확성/정확성, 반드시 종료되는 유한성, 현재 실현 가능한 범위의 유효성.유한성은 종료 여부, 유효성은 실행 가능한 명령인지 여부.
복잡도시간 복잡도는 실행 속도 또는 연산 횟수, 공간 복잡도는 실행 중 동적으로 할당되는 메모리 양.현대 설계에서는 메모리가 커져 공간을 더 쓰고 시간을 줄이는 선택이 자주 쓰임.
Big-OWorst-case. 알고리즘이 가장 운이 없을 때 걸리는 시간의 상한선. 아무리 늦어도 이 안에는 끝난다는 실행 보장을 제공.현업 표준 표기. O(n)의 O는 최악의 경우 상한선을 뜻함.
Big-OmegaBest-case. 가장 운이 좋을 때의 하한선. 최소한 이 정도 효율은 나온다는 증명에 사용.실제 성능 지표로는 Big-O보다 덜 자주 사용.
Big-ThetaAverage-case. 모든 가능한 입력값에 대해 평균적으로 걸리는 시간. 확률 분포 계산이 까다로움.평균이라고 항상 쉽게 계산되는 것이 아님.
복잡도 계산 기준프로그래밍 관점에서 핵심 연산, 가장 오래 걸리는 부분, 가장 많이 반복되는 부분을 기준으로 계산.if 조건의 연산 시간이 제일 길다면 그 연산을 기준으로 복잡도를 계산할 수 있음.

녹취 기반 보강: W2 파이썬 기초와 복잡도 연결

보강 항목녹취 요약 내용기존 내용에 더할 시험 포인트주의
변수와 자료형변수는 데이터를 담는 이름이며 숫자형, 문자형, 불리언, 리스트, 튜플, 딕셔너리 같은 자료형을 다룸.알고리즘 코드의 `arr`, `sum`, `key`, `pivot` 같은 변수는 단순 이름이 아니라 자료형과 가능한 연산을 결정함.예약어를 변수명으로 쓰지 않기.
문자열과 리스트리스트는 인덱싱 후 수정 가능하지만 문자열은 불변이라는 점이 강조됨.정렬 코드는 리스트의 원소 교환과 이동을 전제로 하므로 문자열처럼 불변인 자료와 다르게 다뤄야 함.문자열을 정렬하려면 보통 리스트 변환 또는 새 문자열 생성이 필요.
인덱싱과 슬라이싱0부터 시작하는 인덱스, 마지막 인덱스 -1, 슬라이싱의 끝 범위 제외가 언급됨.Merge Sort의 `arr[:mid]`, `arr[mid:]`와 Quick Sort의 리스트 분할을 이해하는 기초.슬라이싱 끝 인덱스는 포함되지 않음.
연산자`%`는 나머지, `//`는 몫. 예: 7 % 3 = 1, 7 // 3 = 2.`mid = len(arr) // 2`에서 정수 나눗셈을 쓰는 이유와 연결.일반 나눗셈 `/`와 정수 나눗셈 `//`를 구분.
조건문조건문은 첫 번째 `if`부터 순서대로 판단한다는 점이 강조됨.Quick Sort의 `if`, `elif`, `else` 분기와 Heapify의 조건문 순서를 이해하는 데 중요.먼저 참이 된 분기 이후의 `elif`는 실행되지 않음.
반복문반복 가능한 객체와 반복 횟수는 복잡도 계산에 직접 영향을 줌.중첩 반복문은 내부 반복 횟수까지 곱해 계산해야 함.코드 줄 수보다 반복 실행 횟수를 봐야 함.

자료구조.pdf: 메모리 구조와 자료 저장 방식

주제핵심 내용시험 포인트
자료구조 정의컴퓨터는 데이터를 RAM에 저장하며, 데이터를 저장할 때 데이터의 순서나 위치 관계를 규정한 것이 자료구조.효율적인 자료구조는 메모리 사용 효율을 높임.
선형 자료구조데이터가 앞뒤로 1:1 관계를 맺으며 일렬로 늘어선 구조. Array List와 Linked List가 대표적.파이썬 List와 자료구조 용어의 List는 다를 수 있음.
Linked List데이터가 여기저기 흩어져 저장되고 포인터/주소로 연결됨. 특정 데이터 접근은 어렵지만 추가/삭제가 용이.탐색 O(n), 추가/삭제 O(1)로 정리.
Array List데이터가 연속된 영역에 저장되어 주소/포인터로 특정 데이터 접근이 쉬움. 추가/삭제 시 이동이 필요해 시간이 소요.탐색 O(1), 추가/삭제 O(n).
StackLIFO 또는 FILO. push는 넣기, pop은 꺼내기.Last In First Out, First In Last Out의 의미를 정확히 연결.
QueueFIFO. Enqueue는 넣기, Dequeue는 꺼내기. Rear/Tail은 삽입 위치, Front/Head는 삭제 위치.Front가 들어오는 곳이 아니라 나가는 곳. STT 혼선 가능성이 있어 PDF 정의를 기준으로 함.

Sorting_1: 기본 정렬, 분할 정복, Merge Sort

주제핵심 내용교수자 강조점/시험 포인트
정렬 정의데이터를 특정 기준에 따라 오름차순 또는 내림차순으로 나열하는 것. 목적은 이진 탐색 등 효율적 탐색을 가능하게 하는 것.정렬은 레코드의 특정 key value 기준으로 재배열하는 것.
평가 기준시간 복잡도, 공간 복잡도, 안정성. 안정성은 동일 값의 상대적 순서 유지 여부.Stability 정의는 짧지만 시험에 나오기 쉬움.
기본 정렬Bubble, Selection, Insertion. 구현은 쉽지만 중첩 루프 기반이라 일반적으로 O(n^2).비교 횟수는 1부터 n-1까지의 합 = n(n-1)/2 = 1/2 n^2 = O(n^2).
Bubble Sort인접한 두 숫자를 비교하여 앞 값이 크면 교체하는 작업 반복.내부 루프 범위 `range(n - i - 1)`.
Selection Sort미탐색 영역의 최소값을 찾아 가장 왼쪽 값과 교체.`min_idx = i`, `min_idx = j`, 내부 루프 후 1회 swap.
Insertion Sort미탐색 영역 첫 숫자를 왼쪽 정렬 영역과 비교하며 적절한 위치에 삽입.`while j >= 0 and arr[j] > key`와 shift 구조.
Divide and Conquer분할, 정복, 결합. 문제를 Base Case까지 나누고, 하위 문제를 해결한 뒤 답을 합침.학생회장 설문 분배 예시로 큰 문제를 작은 역할 단위로 나누는 원리를 설명.
RecurrenceT(n) = aT(n/b) + f(n). a는 재귀 호출 개수, b는 문제 크기를 나누는 비율, f(n)은 재귀 외 추가 작업.a>b, a<b, a=b의 구조 차이와 예시.
Recursion점화식은 수학적 관계식, 재귀는 이를 컴퓨터로 구현하는 방법론.Base Case와 Recursive Step이 없으면 재귀가 아님.
Merge Sort리스트를 같은 길이로 반복 분할하고, 숫자가 1개가 되면 두 리스트씩 정렬하며 통합.T(n)=2T(n/2)+O(n), 최선/평균/최악 O(n log n), 공간 O(n).

Sorting_2: Quick, Heap, TimSort

Quick Sort피벗을 임의로 하나 선택하고, 피벗보다 작은 요소와 큰 요소로 분할. 원소가 1개가 되면 통합.평균/최선 Θ(n log n), 최악 O(n^2). 피벗 선택이 성능을 좌우.
Quick 연산 횟수[16, 4, 8, 2, 11] 예시의 균형 분할은 피벗 포함 9회, 피벗 미포함 6회로 설명. 정렬된 상태에서 한쪽으로 쏠리면 피벗 포함 14회, 미포함 10회.최악은 T(n)=T(n-1)+n 구조로 O(n^2).
Tree루트, 부모/자식, 형제, 리프, edge, 깊이, 높이, 내부 노드, 차수, 서브트리 등 용어 정리.노드가 N개이면 간선은 N-1개. 사이클 없음.
Binary Tree각 노드가 최소 0개에서 최대 2개의 자식 노드만 가질 수 있는 트리. 왼쪽/오른쪽 자식은 구분됨.이진 트리는 트리지만 1차원 배열로 표현 가능.
Binary Tree 종류정 이진 트리, 완전 이진 트리, 포화 이진 트리, 균형 이진 트리.Heap은 완전 이진 트리 기반.
Heap Tree완전 이진 트리 형태를 유지하면서 부모-자식 관계를 만족. Max Heap은 부모가 자식보다 크거나 같고, Min Heap은 작거나 같음.Max Heap 루트는 최대값.
Heap SortHeap 구조를 만들고, 최대값을 꺼내 배열 맨 뒤로 보낸 뒤 heapify로 재조정 반복.heapify의 숨은 Base Case는 `largest == i`.
성능 측정n=10000, 10회 반복 예시에서 평균: Merge 약 0.00998초, Quick 약 0.00637초, Heap 약 0.01483초, Bubble 약 2.152초, Selection 약 1.176초, Insertion 약 1.086초.Big-O만 같아도 실제 상수, 데이터 상태, 메모리 접근 차이로 성능이 다를 수 있음.
TimSort현실 데이터가 어느 정도 정렬되어 있다는 점에 착안한 하이브리드 정렬. Merge Sort의 안정성과 Insertion Sort의 작은 데이터셋 효율성을 결합.Python, Java, Android, Swift 등에서 채택. 최선 O(n), 평균/최악 O(n log n), 공간 O(n), Stable.

hash_table: Hash Table과 Collision

주제핵심 내용시험 포인트
Hash 의미사전적으로 잘게 썰다 또는 뒤섞다. 알고리즘에서는 임의 길이 input data를 복잡한 규칙으로 분해/혼합해 고정 길이 데이터로 변환하는 과정.Hash Brown 예시로 의미를 설명.
Hash 특징눈사태 효과, 단방향성, 결정론적 특성.입력이 같으면 항상 같은 output. 결과로 원래 input을 역추적하기 어려움.
Direct-address table키를 배열 index로 그대로 사용하는 원시적 구조. 가능한 key의 전체 집합이 작을 때 잘 동작.O(1) 접근이 가능하지만 RAM 낭비가 심함.
Hash Table키를 직접 index로 쓰지 않고 Hash Function을 통해 index를 계산하여 저장 공간 요구량을 줄임.문자와 숫자 등 다양한 key를 사용할 수 있음.
CollisionHash index 결과 개수는 전체 key 가능 개수보다 작기 때문에 서로 다른 key가 같은 index를 가질 수 있음.핵심은 충돌을 적게 만들고, 발생한 충돌을 효율적으로 처리하는 것.
Chaining같은 hash index는 같은 bucket에 저장. 실제 RAM의 같은 공간에 모여 있다는 뜻은 아니며 노드 주소로 연결될 수 있음.탐색은 table[index] 접근 O(1) 후 bucket 내부 탐색 O(k).
Open Addressing고정된 배열 안에서 시작 위치로부터 다른 빈 index를 찾는 방식. Linear, Quadratic, Double Hashing.외부 리스트를 쓰지 않으며 hash 함수 자체가 바뀌는 것은 아님.
Linear Probing충돌 횟수 i에 따라 한 칸씩 이동. 구현이 쉽고 Cache locality가 좋지만 Primary Clustering 문제가 있음.같은 군집을 길게 훑어야 할 수 있음.
Quadratic Probing충돌 횟수의 제곱 값만큼 이동. Linear보다 cluster가 덜하지만 여전히 문제가 남음.자료에는 "Quardratic" 오탈자 형태가 보이나 Quadratic이 표준 철자.
Double Hashing추가 hash 함수를 사용해 key마다 probing 이동 위치를 다르게 부여.두 번째 hash 함수의 결과는 0이면 안 됨.
Resize & RehashingLoad Factor α = n/m이 임계값을 넘으면 배열 크기를 늘리고 hash index를 다시 계산.Resize와 Rehashing은 함께 이루어져야 함. hash function이 배열 크기 정보를 가지고 계산되기 때문.

코드 8: Quick Sort 
def quick_sort(arr):

    if len(arr) <= 1:
        return arr

    pivot = arr[len(arr) // 2]

    left = []
    middle = []
    right = []

    for x in arr:
        if x < pivot:
            left.append(x)
        elif x == pivot:
            middle.append(x)
        else:
            right.append(x)

    return quick_sort(left) + middle + quick_sort(right)

rand_list = [16, 4, 8, 2, 11]  # [보충 설명] 원자료의 rand_list 값은 비어 있어 실행용 예시를 넣음

코드 9: Heapify 
def heapify(arr, n, i):
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2

    if left < n and arr[left] > arr[largest]:
        largest = left

    if right < n and arr[right] > arr[largest]:
        largest = right

    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)

왼쪽/오른쪽 자식과 부모를 비교해 가장 큰 값의 index를 찾습니다. `largest == i`이면 아무 것도 하지 않고 함수가 끝나며, 이것이 숨겨진 Base Case입니다.

코드 10: Heap Sort 
def heap_sort(arr):
    arr = arr.copy()
    n = len(arr)

    # 최대 힙 만들기
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)

    # 힙에서 하나씩 꺼내 정렬
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]
        heapify(arr, i, 0)

    return arr

추가 노트: Huffman Coding 핵심 코드 
import heapq
from itertools import count

def huffman_coding(freq):
    unique = count()
    heap = []

    for char, frequency in freq.items():
        heapq.heappush(heap, [frequency, next(unique), [[char, ""]]])

    if len(heap) == 1:
        only_char = heap[0][2][0][0]
        return {only_char: "0"}

    while len(heap) > 1:
        low1 = heapq.heappop(heap)
        low2 = heapq.heappop(heap)

        for pair in low1[2]:
            pair[1] = "0" + pair[1]
        for pair in low2[2]:
            pair[1] = "1" + pair[1]

        heapq.heappush(heap, [low1[0] + low2[0], next(unique), low1[2] + low2[2]])

    return dict(heap[0][2])

빈도가 가장 낮은 두 노드를 Min Heap에서 꺼내 합치는 과정을 반복합니다. 자주 등장하는 문자는 짧은 코드, 드문 문자는 긴 코드를 받습니다.

  3. j=2: 5와 2 비교. swap → [1, 4, 2, 5]. 가장 큰 5가 맨 뒤 확정.
  4. i=1부터는 range(n - i - 1)로 맨 뒤 확정 구간을 제외하고 반복.
  5. 핵심: 인접 비교를 반복해 큰 값이 뒤로 떠오르므로 Bubble Sort.

Selection Sort 예시: [5, 1, 4, 2]

  1. i=0: 미정렬 구간 첫 위치를 기준으로 min_idx=0.
  2. j=1: 1 < 5이므로 min_idx=1로 갱신.
  3. j=2,3: 4와 2는 현재 최소값 1보다 크므로 유지.
  4. 내부 루프 종료 후 arr[0]과 arr[1]을 1회 swap → [1, 5, 4, 2].
  5. 핵심: 비교는 여러 번 하지만 swap은 한 회전당 마지막에 한 번.

Insertion Sort 예시: [5, 1, 4, 2]

  1. i=1: key=1, 왼쪽 정렬 구간은 [5].
  2. j=0: 5 > 1이므로 5를 오른쪽으로 이동 → [5, 5, 4, 2].
  3. j=-1이 되어 while 종료, arr[j+1] 즉 0번 자리에 key 삽입 → [1, 5, 4, 2].
  4. i=2: key=4, 5를 한 칸 밀고 4 삽입 → [1, 4, 5, 2].
  5. 핵심: swap이 아니라 큰 값을 뒤로 밀어 빈자리에 key를 꽂는 구조.

Merge Sort 예시: [5, 1, 4, 2]

  1. mid=2로 [5, 1]과 [4, 2]로 분할.
  2. [5, 1]은 [5]와 [1]로, [4, 2]는 [4]와 [2]로 다시 분할.
  3. 원소가 1개이면 Base Case이므로 그대로 반환.
  4. merge([5], [1]) → [1, 5], merge([4], [2]) → [2, 4].
  5. 마지막으로 merge([1, 5], [2, 4]) → [1, 2, 4, 5].
  6. 핵심: 분할 자체가 정렬을 끝내는 것이 아니라, 올라오면서 merge가 정렬을 만든다.

Quick Sort 예시: [3, 2, -6, 1, 8, 5, 9, 6]

  1. 피벗을 3으로 보면, 3보다 작은 값은 [2, -6, 1], 같은 값은 [3], 큰 값은 [8, 5, 9, 6].
  2. 왼쪽 [2, -6, 1]과 오른쪽 [8, 5, 9, 6]을 같은 방식으로 재귀 정렬.
  3. 각 부분 리스트가 길이 1 이하가 되면 Base Case로 반환.
  4. 반환 시 quick_sort(left) + middle + quick_sort(right)로 결합.
  5. 핵심: 피벗이 균형을 만들면 빠르고, 한쪽으로 쏠리면 O(n^2)까지 나빠진다.

Heapify 예시: [2, 4, 8, 16, 11]

  1. i=1이면 왼쪽 자식은 3번(16), 오른쪽 자식은 4번(11).
  2. 부모 4보다 왼쪽 자식 16이 크므로 largest=3.
  3. 오른쪽 자식 11은 16보다 작으므로 largest 유지.
  4. largest != i이므로 4와 16을 swap → [2, 16, 8, 4, 11].
  5. 바뀐 위치 3에서 다시 heapify(arr, n, 3). 더 큰 자식이 없으면 자동 종료.
  6. 핵심: 숨은 종료 조건은 더 이상 swap할 필요가 없는 largest == i.

Heap Sort 예시 흐름

  1. 배열의 마지막 부모 n // 2 - 1부터 0까지 내려오며 Max Heap을 만든다.
  2. Max Heap의 루트 0번에는 현재 가장 큰 값이 온다.
  3. arr[0]과 arr[i]를 swap해 최대값을 정렬 완료 구간인 맨 뒤로 보낸다.
  4. 힙 크기를 i로 줄이고 heapify(arr, i, 0)으로 루트부터 재조정한다.
  5. 핵심: 정렬 완료 구간은 다시 heapify 대상에 포함하지 않는다.

Greedy 거스름돈 예시: amount=1600

  1. 동전 [500, 100, 50, 10]을 내림차순 정렬.
  2. 500원: 1600 // 500 = 3, 남은 금액 1600 % 500 = 100.
  3. 100원: 100 // 100 = 1, 남은 금액 0.
  4. 50원, 10원은 더 이상 필요 없으므로 0개.
  5. 결과는 [3, 1, 0, 0]. 단, 동전 체계가 바뀌면 Greedy가 최적이 아닐 수 있다.

DP factorial(5) 예시

  1. 초기 배열 val = [1, 1, 1, 1, 1, 1].
  2. i=1: val[1] = val[0] * 1 = 1.
  3. i=2: val[2] = val[1] * 2 = 2.
  4. i=3: val[3] = val[2] * 3 = 6.
  5. i=4: 24, i=5: 120. 저장된 이전 값을 재사용한다.
  6. 핵심: 재귀처럼 다시 내려가지 않고, 작은 문제부터 표를 채운다.

Huffman Coding 예시 첫 단계

  1. 빈도: A=45, B=13, C=12, D=16, E=9, F=5.
  2. Min Heap에서 가장 낮은 F(5), E(9)를 꺼내 14로 합친다.
  3. 다음으로 C(12), B(13)을 꺼내 25로 합친다.
  4. 이 과정을 반복해 root 빈도 100의 트리를 만든다.
  5. 왼쪽/오른쪽 가지에 0/1을 붙이며 문자별 가변 길이 코드를 만든다.
  6. 핵심: 빈도가 높은 A는 짧은 코드, 빈도가 낮은 F/E는 긴 코드를 받는다.

객관식 함정 포인트

  • Queue에서 Rear/Tail은 들어오는 곳, Front/Head는 나가는 곳.
  • Big-O는 평균이 아니라 최악의 경우 상한선.
  • Direct-address table은 빠르지만 RAM 낭비가 크다.
  • Double Hashing의 두 번째 hash 함수 결과는 0이면 안 된다.
  • TimSort는 Merge와 Insertion을 결합한 Stable 정렬이다.
  • Heap Sort는 Divide and Conquer 개념으로 설명되기보다는 Heap 자료구조와 heapify가 핵심이다.
  • Greedy는 항상 최적해를 보장하지 않는다. 동전 체계 반례처럼 조건에 따라 실패할 수 있다.
  • DP는 미래를 예측하는 방식이 아니라, 과거에 계산한 부분 문제 해를 저장하고 현재에서 조합하는 방식이다.
  • Backtracking과 Branch and Bound는 둘 다 Pruning을 쓰지만, 하나는 조건 만족 해 탐색이고 다른 하나는 최적해 탐색이다.
  • Huffman Coding은 Max Heap이 아니라 빈도수가 낮은 노드를 먼저 꺼내는 Min Heap 전략과 연결된다.

1Big-O, Big-Omega, Big-Theta를 비교 설명하시오.Big-O는 최악 상한, Big-Omega는 최선 하한, Big-Theta는 평균/타이트한 성장률. 현업에서는 실행 보장을 위해 Big-O가 널리 쓰임.
2분할 정복과 재귀의 관계를 Merge Sort를 예로 들어 설명하시오.분할 정복은 설계 기법, 재귀는 구현 방법. Merge Sort는 mid로 left/right를 나누고 Base Case에서 반환한 뒤 merge로 결합.
3Quick Sort의 평균 성능과 최악 성능이 달라지는 이유를 설명하시오.pivot이 균형을 만들면 Θ(n log n), 한쪽으로 치우치면 T(n)=T(n-1)+n으로 O(n^2). 성능은 pivot 선택 로직에 좌우.
4Heap Sort에서 heapify의 역할을 코드 변수와 함께 설명하시오.arr은 heap이 저장된 배열, n은 heap 영역 크기, i는 조정 시작 index. left/right 자식과 비교해 largest를 갱신하고 swap 후 재귀 호출.
5Hash Table에서 Resize와 Rehashing이 반드시 같이 이루어져야 하는 이유를 설명하시오.Hash Function은 배열 크기 정보를 가지고 index를 계산한다. Resize로 m이 바뀌면 기존 key의 index도 달라질 수 있어 Rehashing이 필요.
6Greedy와 Dynamic Programming의 차이를 설명하시오.Greedy는 현재 기준 최선을 즉시 선택하고 과거 해를 저장하지 않는다. DP는 가능한 선택을 고려하고 과거 부분 문제 해를 저장해 현재 최적해를 조합한다. Greedy는 빠르지만 최적해 보장이 어려울 수 있고, DP는 공간을 더 쓰지만 최적해 보장이 가능하다.
7Huffman Coding이 Greedy와 Heap을 어떻게 활용하는지 설명하시오.문자 빈도에 따라 Min Heap에 노드를 넣고, 빈도수가 가장 낮은 두 노드를 반복적으로 꺼내 합친다. 이 과정으로 트리를 만들고, 자주 나오는 문자는 짧은 코드, 드문 문자는 긴 코드를 받아 평균 비트 수를 줄인다.

시험 직전 3분 요약

  • 알고리즘은 입력/출력, 명확성/정확성, 유한성, 유효성을 만족해야 한다.
  • Big-O는 최악의 경우 상한선이며, 최고차항만 남겨 O(n), O(n^2), O(n log n)으로 표현한다.
  • Array는 접근이 빠르고 Linked List는 추가/삭제가 빠르다. Stack은 LIFO, Queue는 FIFO다.
  • Bubble/Selection/Insertion은 O(n^2)이며, 빈칸은 반복 범위, 조건문, swap/shift가 중심이다.
  • Merge Sort는 T(n)=2T(n/2)+O(n), Quick Sort는 pivot 선택에 따라 평균 Θ(n log n)과 최악 O(n^2)이 갈린다.
  • Heap Sort는 완전 이진 트리와 Max Heap, heapify가 핵심이며 left=2*i+1, right=2*i+2를 외운다.
  • Hash Table은 key를 hash index로 바꾸어 저장한다. 충돌은 Chaining과 Open Addressing으로 처리한다.
  • Load Factor α=n/m이 임계값을 넘으면 Resize와 Rehashing을 함께 해야 한다.
  • Greedy는 현재 최선 선택을 반복하지만 반례가 있을 수 있어 Greedy Choice Property와 Optimal Substructure를 확인해야 한다.
  • Dynamic Programming은 Optimal Substructure와 Overlapping Subproblems가 있을 때 과거 해를 저장해 현재 최적해를 만든다.
  • Backtracking은 조건 위반 경로를, Branch and Bound는 최적 가능성이 낮은 경로를 Pruning한다.
  • Huffman Coding은 빈도 낮은 두 노드를 Min Heap으로 합치며, 자주 나오는 문자는 짧은 코드를 받는다.
